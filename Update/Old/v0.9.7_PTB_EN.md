# v0.9.7 — PTB Release Notes

> **Public Test Build** — *March 2026*

This PTB focuses on performance, reliability, and internationalization.

---

## Refactoring & Performance

- **Instant Mod Detail Loading** — The mod detail panel now renders immediately with stored information. Conflict checks and file verification are deferred to a background async worker.
- **Optimized Conflict Engine** — Selective conflict checking now only processes enabled mods or the currently selected mod. This significantly improves responsiveness during mod activation.
- **Frontend Modularization** — The JavaScript codebase has been split into focused modules: `api.js`, `components.js`, `utils.js`, `state.js`.
- **No More System Freezes** — Operations are now chunked and limited to ensure your system stays responsive even during heavy file transfers.
- **Safety First** — Improved integrity checks during mod activation ensure game files are properly backed up before being overwritten.

---

## New Performance Engine

- **Disk I/O Limiter** — Configure maximum transfer speeds per disk to prevent lag during mod activation.
- **Storage Manager Modal** — Track disk health, capacity, filesystem type, and cloud/network storage detection.
- **Performance Dashboard (PiP)** — Real-time CPU, RAM, and Disk I/O monitor with session history, **CSV export**, and timeline replay.
- **Smart ETA Calculation** — The floating monitor predicts remaining time based on your configured I/O limits.

---

## Improvements & New Features

### Language System
- **Dynamic Language Discovery** — BMM auto-detects any `.json` file added to `frontend/Lang/`.
- **Unified Selector** — Clean single dropdown for language selection in Settings.
- **FlagCDN Integration** — High-quality flag icons with offline text fallback.
- **New Translation** — Basic German (`de.json`) support added.

### Version & Build System
- **Static Build Date** — Captured at compile time via Rust build script.
- **Dynamic Injection** — Version and build date automatically injected in Credits, Footer, and Titlebar.
- **Intelligent PTB Badging** — The `-PTB` suffix and blue badge adapt dynamically to the app configuration.

---

## Technical Changes

- Moved all hardcoded version strings to dynamic placeholders.
- Robust Rust-side `Lang/` directory detection for both dev and prod environments.
- Color-coded priority indicators in `.MM` lists with priority values on hover.
- Improved translation system — completely translatable settings interface.
