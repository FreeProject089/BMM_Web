# v0.9.6 — PTB Release Notes

> **Public Test Build** — March 2026

---

## Auto-Update System
BMM now includes a built-in update checker powered by the GitHub Releases API.

- **Automatic startup check** — BMM silently queries GitHub 3 seconds after launch. You are only notified when a new version is available.
- **Manual check button** — A new green "Check for Updates" button is available in the sidebar footer and in the Settings page.
- **Settings toggle** — Auto-update checks can be enabled or disabled from Settings → Auto Update.
- **Update modal** — When a new version is detected, a detailed modal displays:
  - Current version vs. latest version
  - Full release notes (rendered Markdown)
  - Direct download link to the `.msi` installer
- **Smart asset detection** — The download button prioritizes `.msi` > `.exe` > `.zip` assets from the release.
- **Graceful error handling** — If no releases are published yet (404), a friendly info message is shown instead of a generic error.

---

## Profile Deletion Safety
Profiles with active mods can no longer be accidentally deleted.

- If mods are still **enabled** in a profile, BMM now prompts:
  > *"X mod(s) are still active in this profile. Disable them all before deleting?"*
- Accepting the prompt will **automatically disable all active mods** and then delete the profile.
- This prevents orphaned mod files in the game directory.

---

## PTB Mode
A new Public Test Build distribution system has been added.

- Controlled by setting `PTB=true` in `app.cfg`.
- On first launch, a themed welcome modal displays the PTB release notes (from `Update_v*_PTB.md`).
- The modal is shown **once per session** — dismissing it prevents re-display until the app is restarted.
- Uses the app's Vanguard Workstation design language with glassmorphism and accent-blue theming.

## Enhanced Logging System
The diagnostic logging system has been significantly improved for better crash analysis and debugging.

### Backend (Rust)
Every key operation now writes to the session log:
- `[PROFILE]` — Create, update, delete, switch active profile
- `[MOD]` — Add, remove, enable (with file count), disable, update metadata, scan, download, toggle all
- `[INTEGRITY]` — Verify integrity checks with issue count
- `[UPDATE]` — Auto-update checks with current version

### Frontend (JavaScript)
New events captured in the session log:
- **Keyboard shortcuts** — Ctrl+N, Ctrl+E, Ctrl+S, Ctrl+F
- **Drag & drop** — File names and count
- **Toggle/checkbox changes** — Element ID and state
- **Select changes** — Element ID and selected value
- **Window focus/blur** — Track when the app gains or loses focus
- **Unhandled errors** — JavaScript errors and unhandled promise rejections are automatically captured
