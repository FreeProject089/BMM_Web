## Système de mise à jour automatique
BMM inclut maintenant un vérificateur de mise à jour intégré alimenté par l'API GitHub Releases.

- **Vérification automatique au démarrage** — BMM interroge silencieusement GitHub 3 secondes après le lancement. Vous n'êtes averti que lorsqu'une nouvelle version est disponible.
- **Bouton de vérification manuelle** — Un nouveau bouton vert "Vérifier les mises à jour" est disponible dans le pied de page de la barre latérale et dans la page des Paramètres.
- **Option des paramètres** — Les vérifications de mise à jour automatique peuvent être activées ou désactivées dans Paramètres → Mise à jour auto.
- **Modale de mise à jour** — Lorsqu'une nouvelle version est détectée, une modale détaillée affiche :
  - La version actuelle vs la dernière version
  - Les notes de mise à jour complètes (Markdown rendu)
  - Un lien de téléchargement direct vers l'installateur `.msi`
- **Détection intelligente d'assets** — Le bouton de téléchargement donne la priorité aux fichiers `.msi` > `.exe` > `.zip` de la release.
- **Gestion gracieuse des erreurs** : Si aucune release n'est encore publiée (404), un message d'information convivial est affiché au lieu d'une erreur générique.

## Sécurité de suppression de profil
Les profils avec des mods actifs ne peuvent plus être supprimés par accident.

- Si des mods sont encore **activés** dans un profil, BMM affiche désormais :
  > *"X mod(s) sont encore actifs dans ce profil. Les désactiver tous avant de supprimer ?"*
- Accepter l'invite **désactivera automatiquement tous les mods actifs** puis supprimera le profil.
- Cela évite les fichiers de mods orphelins dans le répertoire du jeu.

## Mode PTB
Un nouveau système de distribution Public Test Build a été ajouté.

- Contrôlé en réglant `PTB=true` dans `app.cfg`.
- Au premier lancement, une modale de bienvenue à thème affiche les notes de mise à jour du PTB (provenant de `Update_v*_PTB.md`).
- La modale est affichée **une fois par session** — sa fermeture empêche le réaffichage jusqu'au redémarrage de l'application.
- Utilise le langage de design Vanguard Workstation de l'application avec glassmorphisme et thématique bleu d'accentuation.

## Système de journalisation amélioré
Le système de journalisation de diagnostic a été considérablement amélioré pour une meilleure analyse des crashs et du débogage.

### Backend (Rust)
Chaque opération clé s'écrit désormais dans le journal de session :
- `[PROFILE]` — Création, mise à jour, suppression, changement de profil actif
- `[MOD]` — Ajout, suppression, activation (avec nombre de fichiers), désactivation, mise à jour des métadonnées, scan, téléchargement, bascule groupée
- `[INTEGRITY]` — Vérification de l'intégrité avec décompte des problèmes
- `[UPDATE]` — Vérification de mise à jour auto avec version actuelle

### Frontend (JavaScript)
Nouveaux événements capturés dans le journal de session :
- **Raccourcis clavier** — Ctrl+N, Ctrl+E, Ctrl+S, Ctrl+F
- **Glisser-Déposer** — Noms de fichiers et nombre
- **Changements de bascules/cases** — ID de l'élément et état
- **Changements de sélections** — ID de l'élément et valeur sélectionnée
- **Focus/Blur de la fenêtre** — Suivre quand l'application prend ou perd le focus
- **Erreurs non gérées** — Les erreurs JavaScript et les rejets de promesses non gérés sont automatiquement capturés
